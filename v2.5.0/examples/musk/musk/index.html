Musk

Musk dataset is a classic MIL problem of the field, introduced in Thomas G. Dietterich , Richard H. Lathrop , Tomás Lozano-Pérez (1997). Below we demonstrate how to solve this problem using Mill.jl. The full example is also accessible here, as well as a Julia environment to run it.

For the demo, we load all dependencies:

using FileIO, JLD2, Statistics, Mill, Flux
using Flux: throttle, @epochs
using Mill: reflectinmodel
using Base.Iterators: repeated

and then load the dataset and transform it into a Mill.jl structure. The musk.jld2 file contains:

The resulting BagNode is a structure which holds (i) feature matrix and (ii) ranges identifying which columns in the feature matrix each bag spans. This representation ensures that feed-forward networks do not need to deal with bag boundaries and always process full continuous matrices:

julia> ds = BagNode(ArrayNode(fMat), bagids)     # create a BagNode dataset
BagNode with 92 obs
  └── ArrayNode(166×476 Array with Float32 elements) with 476 obs

Once the data are in Mill.jl internal format, we will manually create a model. BagModel is designed to implement a basic multi-instance learning model utilizing two feed-forward networks with an aggregaton operator in between:

julia> model = BagModel(
           ArrayModel(Dense(166, 10, Flux.tanh)),                      # model on the level of Flows
           meanmax_aggregation(10),                                    # aggregation
           ArrayModel(Chain(Dense(21, 10, Flux.tanh), Dense(10, 2))))  # model on the level of bags
BagModel … ↦ ⟨SegmentedMean(10), SegmentedMax(10)⟩ ↦ ArrayModel(Chain(Dense(21, 10, tanh), Dense(10, 2)))
  └── ArrayModel(Dense(166, 10, tanh))

Instances are first passed through a single layer with 10 neurons (input dimension is 166) with tanh non-linearity, then we use mean and max aggregation functions simultaneously (for some problems, max is better then mean, therefore we use both), and then we use one layer with 10 neurons and tanh nonlinearity followed by linear layer with 2 neurons (output dimension).

Let's check that forward pass works:

julia> model(ds)
2×92 ArrayNode{Matrix{Float32}, Nothing}:
 -1.1352562   -0.5154003  -0.45760936  …  -0.8627351  -1.5617254  -0.86496043
  0.19459999   0.6092025   0.43866342      1.4347763   0.9022629   1.7771704

Since Mill.jl is entirely compatible with Flux.jl, we can use its cross-entropy loss function:

julia> loss(ds, y_oh) = Flux.logitcrossentropy(model(ds) |> Mill.data, y_oh)
loss (generic function with 1 method)

and run simple training procedure using its tooling:

julia> opt = Flux.ADAM()
ADAM(0.001, (0.9, 0.999), IdDict{Any, Any}())

julia> @epochs 10 begin
           Flux.train!(loss, params(model), repeated((ds, y_oh), 1000), opt)
           println(loss(ds, y_oh))
       end
